# 舞台队形编排工具

[English](README.md) | **[▶ 在线演示](https://xpanflow.github.io/stage-formation/)**

![舞台队形编排工具](doc/ScreenShot.png)

一款纯前端单页应用，专为儿童演出设计，支持舞台队形的创建、编辑与动画演示。

## 功能特性

- **真实舞台尺寸** — 以米为单位定义舞台宽度和深度，所有位置坐标均以米存储
- **背景图片** — 支持上传 PNG / JPG / WEBP 格式的舞台照片，自动适配画布
- **演员管理** — 为每位演员设置姓名、颜色和类型（儿童 / 成人 / 教师）
- **等比例渲染** — 演员大小随舞台尺寸和缩放级别自动调整
- **拖拽操作** — 流畅拖动，限制在舞台范围内；支持对齐网格
- **多场景管理** — 创建、重命名、复制、删除场景，每个场景独立保存演员位置
- **动画过渡** — 播放场景间的缓动过渡动画，并预览移动路径
- **缩放与平移** — 鼠标滚轮缩放、双指捏合缩放（触控）、中键 / 空格+拖拽平移
- **测量工具** — 点击两点即可显示实际距离（米）
- **比例标尺** — 舞台上始终显示 1 米参考刻度
- **场景备注** — 每个场景独立的自由文本备注栏（PowerPoint 风格，位于舞台区底部）；自动保存，复制场景时一并保留，JSON 导出/导入均包含备注内容
- **JSON 导出 / 导入** — 将完整项目保存为格式化 `.json` 文件（背景图片不包含）；导入时验证结构并详细报错
- **导出 PNG** — 将当前场景（含背景、演员、网格、标尺）导出为 PNG 图片
- **撤销 / 重做** — 完整操作历史（最多 60 步）
- **自动保存** — 所有数据保存至 `localStorage`，刷新后自动恢复
- **全屏模式** — 一键进入全屏

## 快速开始

```bash
# 直接在浏览器中打开
open index.html

# 或使用本地静态服务器
npx serve .
python3 -m http.server 8080
```

也可以直接访问 **[GitHub Pages 在线演示](https://xpanflow.github.io/stage-formation/)**，无需安装任何东西。

| 按键 | 功能 |
|------|------|
| `Ctrl/Cmd + Z` | 撤销 |
| `Ctrl/Cmd + Y` | 重做 |
| `F` | 舞台适应窗口 |
| `G` | 切换网格显示 |
| `空格 + 拖拽` | 平移舞台 |
| `Delete / Backspace` | 删除选中演员 |
| `Escape` | 取消选中 / 退出测量模式 |

## 使用流程

1. 在右侧面板设置**舞台尺寸**，例如宽 10 米 × 深 6 米
2. 可选：**上传背景**舞台照片
3. 点击左侧面板的 **+ Add** 按钮添加演员
4. **拖动演员**到舞台上，完成第一个场景（Scene 1）的站位编排
5. **添加第二个场景**，位置将从上一场景复制为起点
6. 为第二个场景重新排列演员站位
7. 在底部**场景备注栏**记录编排构思
8. 点击 **▶ Play** 预览过渡动画效果
9. **导出 JSON** 保存项目；按需为每个场景**导出 PNG**

## 只读预览分享

将项目以只读链接分享给任何人，无需账号。

**操作流程：**

1. 点击工具栏 **↓ Export** 导出项目为 `.json` 文件（不含背景图）
2. 将 JSON 文件上传至公开可访问的 URL（如 Cloudflare R2 公开 Bucket、GitHub Gist raw 等）
3. 将 `?load=<json地址>` 拼接到在线演示链接后，发送给对方：

```
https://xpanflow.github.io/stage-formation/?load=https://pub-xxx.r2.dev/projects/my-show.json
```

对方打开链接后，项目自动加载，进入**只读预览模式**（不会覆盖本地存储）。可选择点击 **"保存到本地"** 将其导入自己的工作区。

> **CORS 说明：** JSON 文件的托管服务需允许来自 `https://xpanflow.github.io` 的 `GET` 请求。使用 Cloudflare R2 时，在 Bucket 的 CORS 设置中添加对应的 `AllowedOrigins` 规则即可。

## 部署到 GitHub Pages

将仓库推送至 GitHub，在仓库设置中开启 Pages，选择 `main` 分支根目录即可。无需构建步骤。
